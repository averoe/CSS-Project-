Core theory (landing)

Hash functions

Deterministic, fixed-length digest of arbitrary input. One-way and fast to compute.

  • Preimage resistance: hard to find a message for a given hash.
  • Second preimage: hard to find another message with the same hash.
  • Collision resistance: hard to find any two messages with the same hash.

Avalanche effect

A tiny input change yields a drastically different digest. Small mutations destroy predictability.

  • Integrity breaks are obvious: any tampering changes the digest.
  • No structure leaks: output appears random.

Integrity checks

Store or transmit a hash alongside data. Recompute and compare to detect tampering.

  • Use SHA-256/512 for modern integrity; avoid MD5 for collision risk.
  • For authenticity, pair with signatures or HMAC.

Takeaways
  • Collision resistance: modern hashes (SHA-256/512) make finding collisions computationally infeasible.
  • Avalanche: single-bit flips scatter across the digest; integrity breaks are obvious.
  • Integrity vs authenticity: hashes detect change, but do not prove who computed them; pair with HMAC or signatures.
  • MD5/SHA-1: known collision attacks; avoid for integrity protection.
